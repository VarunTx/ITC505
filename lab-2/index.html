1. Abstract

Unmanned aerial vehicles (UAVs) rely significantly on their propulsion systems, with the motor and propeller being vital parts in determining drone performance. This project delves into improving UAV performance and reliability by utilizing machine learning approach. which is intend to optimize drone motor performance by doing an in-depth analysis using regres- sion models. This method uses input variable throttle signal to estimate rotational speed and using rotational speed, voltage and current as input to estimate thrust and torque of multirotor systems, eliminating the need for extra thrust and torque sensors. Furthermore, we provide a strong model capable of reliably forecasting Motor Electrical Speed (RPM), Thrust and Torque from unseen data, Motor Electrical speed predictions had a prediction accuracy of > 97% speed with an error < 2.5% of the data set range for 80% of the dataset and Thrust predictions had a prediction accuracy > 87% thrust with an error < 6% of the data set range for 80% of the dataset while Torque predictions had a prediction accuracy > 99% torque with an error < 1.2% of the data set range for 80% of the dataset . Each dataset is rigorously reviewed to reveal significant information and insights of motor properties. The dataset used in this work was obtained from RCtestBench 1580, which provided valuable information on motor characteristics, electrical signals, and mechanical outputs.

Addendum

As I worked on converting the academic paper into HTML, I encountered several challenges in understanding how to structure the document. Below is a summary of the decisions I made, why I made them, and how I resolved issues I encountered along the way.

Using `article` and `section`: Initially, I wasn’t sure if I should use `div` or `article` to encapsulate the content of the paper. After learning that `article` is meant for self-contained content, I realized it was the best choice for this website. I used the first `article` to hold the main content of the paper and the second `article` for the addendum as both sections are independent but related pieces of content.

Handling the Title and Authors: At first, I used basic `p` tags for the title and author information, but later I discovered that HTML provides the `header` and `address` tags, which are more appropriate for this kind of content. The `header` tag wraps the title of the paper, and the `address` tag is used to hold the author names and contact information. This was a semantic improvement, making the structure clearer. Initially, I also forgot to separate the authors' names into distinct lines, which made it hard to read. I corrected this by placing each author in their own `p` tag under the `address`.

Adding Tables: When I first added the table for the “Overview of Components Used,” I didn’t know how to format the borders. Initially, the table had no structure and was difficult to read. I later added `thead` and `tbody` tags to organize the table content and used CSS to add borders with border: 1px solid black; on the table cells (`th`, `td`). This improved readability significantly.

Citations and References: I struggled with how to represent citations within the text. Initially, I used plain text like [1], but then I realized I could use anchor tags (`a`) to link citations in the text directly to the references at the end of the paper. I used id attributes in the reference list and created links with href="#refX". This improved the interactivity and made it easier to navigate the document.

Problems with Styling: At first, I tried using inline styles to format various elements like headings and tables. This quickly became messy and hard to manage. I later moved to using a `style` tag in the `head` section to consolidate the styling, which made the HTML file much cleaner and easier to maintain. I also realized that using margin-left in the body style made the text more readable by keeping it aligned consistently across the page.
